# سوق التشاليح

تطبيق ويب عربي **RTL** لربط عملاء قطع غيار السيارات بالموردين داخل السعودية، مع قسم مستقل لبيع السيارات. تم بناء الواجهة لتبقى قريبة جداً من المرجع البصري المطلوب، مع الحفاظ على الهيدر الأخضر بعنوان **سوق التشاليح**، وبطاقتي **طلب قطعة** و**بيع سيارة** كما طُلِب.

## نظرة عامة

يعتمد المشروع على واجهة React مع خادم Node/Express وطبقة **tRPC**، بينما تتم إدارة المصادقة عبر **Supabase OTP** لرقم الجوال، وتُحفَظ البيانات في قاعدة البيانات، كما يتم رفع الصور إلى **Supabase Storage**. يدعم التطبيق أدوار العميل والمورد، مع تدفقات إنشاء الطلبات، تقديم العروض، نشر السيارات، وإضافة التقييمات بعد اكتمال الصفقة. كما تم تفعيل **وضع OTP تجريبي** للرقم `0536051509` باستخدام الرمز `252525` لتسهيل المراجعة السريعة.

| العنصر | التفاصيل |
|---|---|
| الواجهة | React 19 + Tailwind CSS 4 |
| الخادم | Express + tRPC |
| قاعدة البيانات | MySQL عبر Drizzle ORM |
| المصادقة | Supabase Phone OTP |
| التخزين | Supabase Storage |
| اللغة والاتجاه | العربية + RTL |

## المميزات الحالية

يوفر المشروع الصفحة الرئيسية بالتصميم العربي المطلوب، مع بطاقات الدخول الرئيسية. كما يتضمن تسجيل الدخول برقم الجوال السعودي، ولوحة عميل لمتابعة الطلبات والعروض، ولوحة مورد للاطلاع على الطلبات المفتوحة وتقديم عروض السعر، بالإضافة إلى نموذج بيع سيارة ونظام تقييم بعد إتمام الصفقة. ولأغراض الاختبار السريع يمكن استخدام وضع OTP التجريبي بالرقم `0536051509` والرمز `252525`.

## تشغيل المشروع محلياً

لبدء التشغيل محلياً، أنشئ ملف `.env` يدوياً اعتماداً على جدول متغيرات البيئة الموضّح في هذا README، ثم ثبّت الاعتماديات وشغّل الخادم التطويري.

```bash
pnpm install
pnpm dev
```

بعد ذلك سيعمل التطبيق محلياً على المنفذ الافتراضي الذي يحدده الخادم.

## متغيرات البيئة

يعتمد المشروع على مجموعة من المتغيرات، أهمها متغيرات Supabase والاتصال بقاعدة البيانات وإعدادات الجلسات.

| المتغير | الوصف |
|---|---|
| `SUPABASE_URL` | رابط مشروع Supabase |
| `SUPABASE_ANON_KEY` | المفتاح العام المستخدم من الواجهة |
| `SUPABASE_SERVICE_ROLE_KEY` | مفتاح الخدمة المستخدم من الخادم لرفع الصور والتحقق |
| `DATABASE_URL` | رابط الاتصال بقاعدة البيانات |
| `JWT_SECRET` | مفتاح توقيع الجلسات |
| `VITE_APP_ID` | معرف التطبيق |
| `OAUTH_SERVER_URL` | رابط خادم OAuth |
| `VITE_OAUTH_PORTAL_URL` | رابط بوابة OAuth للواجهة |
| `OWNER_OPEN_ID` | معرف المالك |
| `OWNER_NAME` | اسم المالك |
| `BUILT_IN_FORGE_API_URL` | رابط خدمات النظام المساعدة |
| `BUILT_IN_FORGE_API_KEY` | مفتاح خدمات النظام المساعدة |
| `VITE_FRONTEND_FORGE_API_URL` | رابط الخدمات المتاح للواجهة |
| `VITE_FRONTEND_FORGE_API_KEY` | مفتاح الخدمات المتاح للواجهة |

## بنية التطبيق الوظيفية

يعمل التطبيق بمنطق بسيط ومباشر. يسجل المستخدم دخوله عبر OTP، ثم يتم جلب حالته الحالية من الخادم. إذا كان دوره عميلاً، يستطيع إرسال طلب قطعة ومتابعة العروض الواردة واعتماد أحدها ثم إتمام الصفقة وإضافة تقييم. وإذا كان دوره مورداً، تظهر له الطلبات المتاحة ويمكنه تقديم عرض سعر مع صور القطعة. كما يتيح التطبيق قسماً منفصلاً لنشر السيارات المعروضة للبيع مع صور متعددة.

## خطوات النشر على Vercel

بما أن هذا المشروع **ليس تطبيق Next.js** بل مشروع React + Express + tRPC، فإن النشر على **Vercel** قد يحتاج عناية إضافية أو تعديلاً في البنية بحسب إعداد مشروعك على Vercel. لذلك يُنصح أولاً برفع المشروع إلى GitHub، ثم ربطه داخل Vercel، مع مراجعة إعدادات البناء والتشغيل بدقة.

### 1) ربط المشروع من GitHub

سجّل الدخول إلى حسابك في Vercel، ثم اختر **Add New Project**، وبعدها اربط المستودع:

`https://github.com/taherhhisam1977/souq-tashalih`

### 2) إعداد أوامر البناء

استخدم الإعدادات التالية كنقطة بداية:

| الإعداد | القيمة |
|---|---|
| Install Command | `pnpm install` |
| Build Command | `pnpm build` |
| Output Directory | `dist/public` أو اتركه فارغاً إذا كنت ستعتمد على إعداد مخصص |

### 3) إضافة متغيرات البيئة في Vercel

أضف جميع المتغيرات المذكورة في قسم **متغيرات البيئة** داخل إعدادات المشروع في Vercel، مع التأكد من وضع القيم الصحيحة لمفاتيح Supabase وقاعدة البيانات.

### 4) ملاحظة مهمة حول التوافق

لأن المشروع يستخدم خادماً مخصصاً عبر Express وtRPC، فقد تحتاج إلى **تكييف طريقة التشغيل على Vercel** أو تحويل جزء الخادم إلى Serverless Functions إذا ظهرت مشاكل تشغيل بعد أول نشر. لذلك فإن النشر الخارجي على Vercel قد يتطلب تعديلات إضافية حسب بيئة الحساب وإعدادات المشروع.
